# Market Microstructure Simulation and Optimal Execution Research Framework

**Project:** Q-Micro
**Status:** Research prototype (simulated markets only — no live data feed)

## 1. Methodology

Q-Micro reconstructs core exchange mechanics from first principles rather than
fitting statistical models to historical price series. A price-time priority
matching engine (`core/`) processes LIMIT, MARKET, STOP, and CANCEL orders
against a heap-indexed limit order book. Synthetic order flow is generated by
four agent archetypes — Noise, Informed, Market Maker, Institutional — whose
interaction produces emergent spread, depth, and price dynamics comparable
in structure (not in calibrated magnitude) to real electronic markets.

Microstructure diagnostics (Kyle's lambda, Amihud illiquidity, VPIN, order
flow imbalance) are computed directly from the engine's own order/trade
history, avoiding the endogeneity problems that arise when these measures
are estimated from data the researcher didn't generate.

Execution algorithms (TWAP, VWAP, Almgren-Chriss optimal trajectories, and a
DQN-based RL agent) are evaluated against the same synthetic market using
Implementation Shortfall as the common cost metric, holding the underlying
price process fixed across algorithms via seeded regime replay.

## 2. Architecture

Order → OrderBook (price-time priority) → MatchingEngine → ExchangeSimulator
↑
SyntheticMarketGenerator (agent population)
↓
microstructure/  execution/  strategies/  simulation/  analytics/
↓
dashboard/ (Streamlit terminal)

Core engine has zero external dependencies (pure stdlib), so all downstream
modules can be developed, tested, and version-controlled independently of
`numpy`/`torch` availability.

## 3. Results (illustrative — parameters not yet calibrated to a specific asset)

- Spread widens predictably under `Regime.VOLATILE` vs `Regime.CALM` given
  the reduced-form `spread_model`, consistent with the direction (not
  magnitude) of empirical spread-volatility relationships.
- Kyle's lambda estimated from the synthetic trade tape is strictly positive
  whenever `InformedTrader` agents are active, and approaches zero when the
  population is Noise-Trader-only — a sanity check that informed flow is the
  mechanism generating measurable price impact in this simulator.
- Preliminary DQN execution-cost comparisons against TWAP/VWAP baselines are
  pending a full training run (see `strategies/rl_execution_agent.py::train`)
  and should not yet be interpreted as evidence of outperformance.

## 4. Limitations

- Agent behavior is rule-based/stochastic, not calibrated to real order-flow
  microstructure data (e.g. LOBSTER, NASDAQ ITCH) — dynamics are
  qualitatively, not quantitatively, realistic.
- No latency, colocation, or network-jitter modeling; all orders are
  processed synchronously and instantaneously.
- STOP-order triggering is last-trade-price based, not a full book-sweep
  simulation.
- The RL agent's reward function uses a simplified linear risk-aversion
  term; no formal proof of convergence to the Almgren-Chriss optimal
  trajectory is provided or claimed.
- Results in Section 3 are simulator sanity checks, not backtested
  performance claims — no real capital or live market implications should
  be drawn from them.

## 5. Possible Institutional Applications

- Training/benchmarking ground for execution algorithms before live desk
  deployment, isolating algo logic from live-market noise.
- Educational tool for explaining market-maker inventory risk, adverse
  selection (VPIN), and price impact to non-quant stakeholders.
- Extensible testbed for new microstructure signals prior to costly
  real-data acquisition.
